Build the session monitor frontend in the Linux release jobs so the Linux binaries no longer ship with an empty `packages/monitor/dist/` and `mirrord ui` serves HTML instead of 404. The macOS CLI release jobs were fixed earlier via the xtask path; this extends the same `corepack enable pnpm` + `pnpm run build` approach to the cross-build targets. Windows is intentionally not covered — the local session-monitor API isn't supported on Windows, so its binary never serves the UI.
