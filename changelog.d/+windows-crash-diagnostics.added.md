Added crash and exception diagnostics to the Windows layer so an abnormal exit during layer injection is self-explanatory. A handled crash (such as an access violation) now writes a text crash record with the faulting module and a short stack, plus a minidump, and the injection steps are bracket-logged so a truncated log pinpoints where a process died. The `Process synchronization failed: No init event found` error now reports the process's session role and whether its parent is still alive (with its exit code), distinguishing an internal crash from an external termination such as an `EDR` killing the injector. A per-session out-of-process monitor writes minidumps from outside the crashed process, detects processes that are terminated with no handler firing, and surfaces a short, copyable crash report. An early layer-initialization failure (for example a child stranded because its parent died before initialization finished) is now surfaced through the same report instead of vanishing as a silent exit. The reporter stays silent on a normal exit at any exit code, and only treats genuine native faults and external kills as reportable: an application's own uncaught exception (including a managed .NET exception), a `Ctrl-C`, and an intentional debugger or IDE-extension stop are left alone. Each incident's files share one name stem in the log directory, and the attachable `.zip` holds the report, the crash record, and the layer logs, while the memory dump is kept separate and local because it can contain sensitive in-memory data. A one-line module summary at startup flags any loaded security or endpoint-protection products.
