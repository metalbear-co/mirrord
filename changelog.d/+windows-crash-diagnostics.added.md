Added crash and exception diagnostics to the Windows layer so an abnormal exit during injection is self-explanatory: a native fault or an external kill (such as an `EDR` terminating the injector) now yields a text crash record, a minidump, bracket-logged injection steps, and a short copyable report from a per-session out-of-process monitor, while normal exits and an application's own exceptions (managed .NET, `Ctrl-C`, an intentional debugger stop) are left alone. Each incident's files share one name stem, the attachable `.zip` bundles the report, crash record, and layer logs (the memory dump stays local since it can hold sensitive data), and a startup module summary flags any loaded security or endpoint-protection product.
