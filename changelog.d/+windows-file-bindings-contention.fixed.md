Fixed concurrent file I/O contending the Windows `IOCP` file-binding map under mirrord. Many parallel open/close operations serialized on a single `RwLock`, so `bind` and `unbind` could give up under contention and drop a file's async-read completion binding, or leak its entry on close. The binding now lives in the file's managed-handle record, which shards its lock across independent locks and drops the binding atomically when the handle closes.
