Added an export button to the `mirrord ui` events panel that downloads a single zip with the session log as JSON (session info, config, processes, port subscriptions, and all captured events, with a count of events dropped by the in-memory cap) and a HAR 1.2 file of the HTTP exchanges that can be imported into browser DevTools or replay tools. Incoming HTTP events now also capture request headers, response status and headers, and request/response body previews (bounded by `experimental.session_monitor_body_limit`, default 32KB, `0` disables body capture), all shown in the event detail dialog. Sensitive headers (`Authorization`, `Cookie`, `Set-Cookie`, and similar) are redacted in exports.
